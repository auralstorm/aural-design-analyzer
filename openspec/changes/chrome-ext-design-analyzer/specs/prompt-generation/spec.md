## ADDED Requirements

### Requirement: 生成结构化 Prompt
系统 SHALL 将设计提取和结构分析的结果整合为一份结构化的 Prompt 文本，按预定义模板组织，包含以下分节：设计系统概述、颜色规范、排版规范、间距规范、视觉效果规范、页面结构、组件规范、布局指南。

#### Scenario: 生成完整 Prompt
- **WHEN** 设计提取和结构分析均已完成
- **THEN** 系统生成的 Prompt 包含所有分节，每个分节使用 Markdown 格式，内容具体且可直接用于 AI 对话。Prompt 开头包含一段概述（描述网站的整体设计风格），结尾包含使用建议

#### Scenario: 部分数据缺失时的降级
- **WHEN** 某些分析维度未获取到有效数据（如页面无阴影、无动画）
- **THEN** 系统在 Prompt 中省略对应分节，不输出空白或占位内容，确保 Prompt 简洁有效

### Requirement: Prompt 模板分节结构
系统 SHALL 按以下固定顺序组织 Prompt 分节：

1. 设计风格概述（一段话总结）
2. 颜色规范（色板 + 用途说明）
3. 排版规范（字体层级表）
4. 间距规范（间距比例）
5. 视觉效果（圆角、阴影、动效）
6. 页面结构（语义化结构树）
7. 组件规范（识别到的组件列表及样式）
8. 布局指南（布局模式 + 响应式断点）

#### Scenario: 验证分节完整性
- **WHEN** 所有分析数据均可用
- **THEN** 生成的 Prompt 严格按照上述 8 个分节顺序输出，每个分节有清晰的标题分隔

#### Scenario: 分节内容格式
- **WHEN** 生成颜色规范分节
- **THEN** 每个颜色以 `色名: #HEX (HSL) - 用途描述` 格式输出，按主色→辅助色→强调色→中性色→背景色顺序排列

### Requirement: 一键复制 Prompt
系统 SHALL 在 Popup 界面中提供「复制 Prompt」按钮，点击后将完整的 Prompt 文本复制到系统剪贴板。

#### Scenario: 复制成功
- **WHEN** 用户点击「复制 Prompt」按钮
- **THEN** 完整的 Prompt 文本被复制到剪贴板，按钮显示「已复制」反馈，2 秒后恢复原始状态

#### Scenario: 分析未完成时
- **WHEN** 用户在分析未完成时尝试复制
- **THEN** 按钮显示为禁用状态，提示「请先完成分析」

### Requirement: Popup 分析结果预览
系统 SHALL 在 Popup 界面中展示分析结果的可视化预览，包括颜色色板、字体样本、间距示意。

#### Scenario: 展示颜色色板预览
- **WHEN** 设计提取完成后用户打开 Popup
- **THEN** Popup 顶部展示提取到的颜色色板，每个颜色显示为色块 + HEX 值，按分类分组

#### Scenario: 展示排版预览
- **WHEN** 设计提取完成
- **THEN** Popup 展示字体层级预览，使用实际提取到的字体族和字号渲染示例文本

#### Scenario: 未分析状态
- **WHEN** 用户首次打开 Popup 且未执行过分析
- **THEN** Popup 显示引导界面，包含「分析当前页面」按钮和简要说明

### Requirement: Prompt 可定制性
系统 SHALL 允许用户在 Popup 中选择要包含在 Prompt 中的分节，取消勾选的分节不会出现在最终 Prompt 中。

#### Scenario: 用户取消某些分节
- **WHEN** 用户取消勾选「可访问性」和「响应式断点」分节
- **THEN** 生成的 Prompt 不包含这两个分节，其余分节保持完整

#### Scenario: 全选和全不选
- **WHEN** 用户取消所有分节的勾选
- **THEN** 「复制 Prompt」按钮显示为禁用状态，提示「请至少选择一个分节」
